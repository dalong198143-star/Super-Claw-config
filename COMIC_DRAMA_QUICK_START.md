# 🎭 AI漫剧制作 - 快速上手指南

## 🚀 5分钟快速体验

### 第一步：访问应用

开发服务器已启动，请在浏览器中访问：
- **前端**: http://localhost:5173/

### 第二步：选择工具

在左侧边栏找到 **🎭 AI漫剧** 工具卡片（第一个），点击进入。

---

## 📖 完整使用流程

### 步骤1️⃣：导入剧本

#### 方式A：粘贴文本
1. 在文本框中粘贴你的剧本或小说片段
2. 建议长度：500-2000字符（约1-3个场景）
3. 点击"下一步：生成分镜 →"

**示例剧本**：
```
深夜，急诊室里灯火通明。已经连续工作十个小时的李明医生疲惫地靠在走廊的墙上。

"李医生，3号床病人情况恶化！"护士小王急匆匆跑过来。

李明立刻打起精神，快步走向病房。监护仪发出刺耳的警报声，病人的心跳正在急速下降。

"准备除颤器！"李明大声喊道，额头上渗出细密的汗珠。
```

#### 方式B：上传文件
1. 点击上传区域或拖拽文件
2. 支持格式：TXT、PDF、DOC、DOCX
3. 文件大小限制：< 10MB

---

### 步骤2️⃣：AI生成分镜

1. 点击"🤖 开始AI分析"按钮
2. 等待AI处理（约2秒模拟时间）
3. 查看生成的分镜脚本

**分镜预览包含**：
- 📊 统计面板：场景数、角色数、镜头数
- 🌳 分镜树：每个场景的详细信息
  - 场景ID和描述
  - 镜头列表（景别+台词）

**示例输出**：
```
场景 S001 - 在医院的走廊里，阳光透过窗户...
  ├─ 镜头 #1 [medium_shot] "这已经是今天第几台手术了？..."
  └─ 镜头 #2 [close_up] (无台词)

场景 S002 - 急诊室内，医护人员忙碌地...
  └─ 镜头 #1 [long_shot] "准备除颤器！血压下降！"
```

如果不满意，可以点击"🔄 重新生成"

---

### 步骤3️⃣：配置角色

#### 角色卡片操作
每个识别出的角色会显示为一张卡片：

1. **上传参考图**（可选但推荐）
   - 点击"上传参考图"按钮
   - 选择角色的清晰正面照片
   - 帮助AI保持角色一致性

2. **选择LoRA模型**（可选）
   - 动漫风格 v1 - 适合二次元风格
   - 写实人像 - 适合真实风格
   - Q版角色 - 适合可爱风格

#### 一致性控制方案

系统提供三种方案：

**🛡️ 三保险方案（推荐）**
- 使用：LoRA + IP-Adapter + ControlNet
- 优点：最稳定，角色脸部绝对一致
- 适用：长篇漫剧、商业项目

**⚡ 快速方案**
- 使用：IP-Adapter + 高级提示词
- 优点：速度快，无需训练
- 适用：短篇测试、快速原型

**🎨 基础方案**
- 使用：仅详细提示词
- 优点：最简单
- 适用：学习实验

点击"下一步：生成资产 →"继续

---

### 步骤4️⃣：批量生成图像

#### 界面说明

**顶部进度条**：显示整体生成进度（如 3/8 镜头）

**场景轮播**：
- 左侧显示当前场景标题
- 右侧网格展示该场景的所有镜头

**镜头卡片**：
- 未生成：显示镜头编号和景别标签
- 已生成：显示生成的图像预览

#### 生成方式

**方式A：单个生成**
1. 找到未生成的镜头卡片
2. 点击"🖼️ 生成图像"按钮
3. 等待1-2秒，图像出现

**方式B：批量生成**
1. 点击底部"🚀 批量生成全部"按钮
2. 系统自动按顺序生成所有镜头
3. 观察进度条实时更新

#### 生成完成后
- 确认所有镜头都已生成
- 点击"下一步：合成视频 →"进入最后一步

---

### 步骤5️⃣：最终合成

#### 合成检查清单

系统会自动检查：
- ✓ 分镜脚本（X场景）
- ✓ 角色配置（X角色）
- ✓ 图像资产（X镜头）
- ○ 音频生成（TTS配音 + BGM）
- ○ 字幕生成（SRT格式）
- ○ 视频合成（FFmpeg渲染）

#### 音频配置

**配音引擎选择**：
- Azure TTS (中文) - 推荐，音质好
- ElevenLabs (多语言) - 最自然，价格高
- 本地Coqui TTS - 免费，需自行部署

**背景音乐选择**：
- 自动生成 (AI作曲) - 根据场景情绪匹配
- 上传自定义BGM - 使用自己的音乐文件
- 无背景音乐 - 仅保留对话和音效

#### 开始合成

1. 点击"🎬 开始最终合成"按钮
2. 观察5阶段进度：
   - ✓ 生成TTS配音 (20%)
   - ✓ 添加背景音乐 (40%)
   - ✓ 生成字幕文件 (60%)
   - ✓ FFmpeg视频合成 (80%)
   - ✓ 渲染完成 (100%)

3. 等待完成（模拟时间约30秒）

#### 结果预览

合成成功后显示：
- 🎥 视频播放器（可播放预览）
- 📊 视频信息（时长/场景数/镜头数）
- 📥 下载按钮
- 🔄 重新创作按钮
- ✏️ 修改分镜按钮（返回步骤4调整）

## 💡 最佳实践技巧

### 1. 剧本编写要点

**好的剧本特征**：
- ✅ 场景描述清晰（地点、时间、氛围）
- ✅ 角色动作明确（表情、姿态、移动）
- ✅ 对话简洁有力（避免过长独白）
- ✅ 情绪变化明显（便于AI理解）

**示例对比**：

❌ **差的描述**：
```
两个人在说话。
```

✅ **好的描述**：
```
李明皱着眉头，双手抱胸站在窗前。小王紧张地搓着手，声音颤抖地说："医生，病人可能撑不过今晚了。"
```

### 2. 角色一致性秘诀

**上传参考图技巧**：
- 使用清晰的正面照片
- 光线均匀，无遮挡
- 表情中性（避免夸张表情）
- 分辨率至少512x512

**提示词固定模板**：
```
[角色名], [年龄]yo, [性别], [发型], [服装], [显著特征], [表情], [动作]
```

示例：
```
Li Ming, 30yo, male, short black hair, white doctor coat, stethoscope around neck, tired expression, leaning against wall
```

### 3. 镜头设计原则

**景别搭配**：
- 远景（long_shot）- 建立场景，展示环境
- 中景（medium_shot）- 展示角色互动
- 近景（close_up）- 突出表情和情绪
- 特写（extreme_close_up）- 强调细节

**经典组合**：
```
场景开始：远景（交代环境）
    ↓
角色出场：中景（展示动作）
    ↓
情绪高潮：近景/特写（捕捉表情）
    ↓
场景结束：拉远镜头（过渡到下一场景）
```

### 4. 成本控制策略

**分层生成**：
- 关键镜头（对话/情绪）- 用高质量参数
- 过渡镜头（空镜/远景）- 用快速参数
- 背景镜头（人群/风景）- 用低成本模型

**批量优化**：
- 一次性生成多个镜头享受批量折扣
- 复用已生成的角色LoRA模型
- 缓存常用场景模板

## ❓ 常见问题

### Q1: 为什么生成的角色长得不一样？

**A**: 这是角色一致性问题，解决方案：
1. 上传高质量的参考图
2. 选择合适的LoRA模型
3. 使用"三保险方案"
4. 在提示词中使用固定的角色描述

### Q2: 生成的视频太短怎么办？

**A**: 调整分镜脚本中的duration字段：
- 单个镜头建议2-5秒
- 整个场景建议5-15秒
- 一集漫剧建议30-90秒

### Q3: 如何添加背景音乐？

**A**: 在步骤5的音频配置中选择：
- "自动生成" - AI根据场景情绪作曲
- "上传自定义" - 上传MP3/WAV文件
- 注意版权问题的音乐不能使用

### Q4: 可以导出分镜JSON吗？

**A**: 当前版本暂不支持，后续版本会添加：
- 导出JSON功能
- 导入已有分镜
- 分享分镜模板

### Q5: 生成失败怎么办？

**A**: 检查以下几点：
1. 网络连接是否正常
2. API额度是否充足
3. 提示词是否包含敏感内容
4. 尝试简化描述重新生成

## 🎯 进阶技巧

### 1. 手动编辑分镜JSON

如果你懂技术，可以直接修改生成的JSON：

```json
{
  "scenes": [
    {
      "scene_id": "S001",
      "shots": [
        {
          "shot_id": 1,
          "camera": "close_up",  // 改为特写
          "duration": 5,         // 延长到5秒
          "prompt": "你的自定义提示词"
        }
      ]
    }
  ]
}
```

### 2. 创建分镜模板库

保存成功的分镜配置，下次直接套用：

**浪漫场景模板**：
```json
{
  "template_name": "Romantic Scene",
  "camera_sequence": ["long_shot", "medium_shot", "close_up"],
  "lighting": "soft_warm",
  "color_palette": ["pink", "orange", "gold"]
}
```

### 3. A/B测试不同参数

对同一场景生成多个版本：
- 版本A：使用动漫风格LoRA
- 版本B：使用写实风格LoRA
- 对比效果，选择最佳
