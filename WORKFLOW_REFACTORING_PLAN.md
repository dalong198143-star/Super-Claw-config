# 🎯 混合模式工作流改造方案（方案四）

## 📋 改造目标

1. **移除学习功能**：删除任务系统、学习工作流、提交表单等学习相关功能
2. **工具化改造**：将6个AI创作工具作为核心功能
3. **思维导图式工作流**：实现三种模式的混合工作流系统

---

## 🏗️ 架构设计

### 当前架构问题
- ❌ 包含任务系统（学习任务）
- ❌ 复杂的学习工作流（6步流程）

- ✅ 已有6个工具组件（可复用）

### 第二阶段：实现三种工作流模式（3天）

#### 2.1 新手引导模式（线性流程）
**特点**：步骤清晰，适合初学者

**界面设计**：
```
步骤1: 选择素材
  [上传图片] [上传视频] [文生图]

步骤2: AI处理
  [换装] [重绘] [动作迁移] [图生视频]

步骤3: 后期编辑
  [视频拼接] [放大] [风格转换]

步骤4: 导出成品
  [下载] [分享] [保存到历史]
```

**实现要点**：
- 使用现有的 WorkflowSteps 组件
- 每个步骤显示可用的工具
- 上一步完成后才能进入下一步
- 提供进度指示

#### 2.2 专家自由模式（画布编辑）
**特点**：高度灵活，专业创作者

**界面设计**：
```
┌─────────────────────────────────────┐
│  工具面板          │    画布区域      │
│  ┌─────────────┐  │                 │
│  │ 文生图      │  │  [节点1]──[节点2]│
│  │ 上传图片    │  │     │           │
│  │ 上传视频    │  │  [节点3]         │
│  │ 换装        │  │                 │
│  │ 重绘        │  │                 │
│  │ 动作迁移    │  │                 │
│  │ 图生视频    │  │                 │
│  │ 视频拼接    │  │                 │
│  └─────────────┘  │                 │
└─────────────────────────────────────┘
```

**实现要点**：
- 创建新的 WorkflowCanvas 组件
- 支持拖拽添加节点
- 支持连线定义数据流
- 可保存/加载工作流模板

#### 2.3 智能推荐模式（AI辅助）
**特点**：智能化，降低决策成本

**界面设计**：
```
┌─────────────────────────────────────┐
│  目标输入框                          │
│  "我想制作一个产品展示视频"          │
│  [分析需求]                         │
├─────────────────────────────────────┤
│  推荐工作流:                         │
│  ┌─────────────────────────────┐   │
│  │ 1. 上传产品图片              │   │
│  │ 2. 重绘洗图(优化画质)        │   │
│  │ 3. 图生视频(添加动态效果)    │   │
│  │ 4. 视频拼接(添加背景音乐)    │   │
│  │ 5. 导出成品                  │   │
│  └─────────────────────────────┘   │
│  [应用此工作流] [自定义修改]        │
└─────────────────────────────────────┘
```

**实现要点**：
- 创建 WorkflowRecommender 组件
- 基于关键词匹配预设模板
- 支持用户反馈优化推荐

## ⚠️ 风险评估

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|---------|
| 画布编辑器复杂度高 | 中 | 高 | 先实现基础版，后续迭代 |
| 用户学习成本高 | 低 | 中 | 提供详细教程和示例 |
| 性能问题（多节点） | 低 | 中 | 限制最大节点数（20个） |
| 浏览器兼容性 | 低 | 低 | 使用标准Web API |
